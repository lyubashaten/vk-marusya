# VK Маруся

«VK Маруся» — это бесплатная платформа для поиска и оценки фильмов.

🔍 **Ищите** фильмы по названию  
⭐ **Узнавайте** рейтинг, читайте описание и смотрите трейлеры  
❤️ **Сохраняйте** понравившиеся фильмы в избранное и возвращайтесь к ним позже

Платформа подойдёт всем, кто хочет вести свой кинодневник и не забывать, что посмотреть вечером.

## 🛠 Стек технологий

| Категория | Технологии |
|-----------|------------|
| **Фреймворк** | Vue 3 (Composition API, `<script setup>`) |
| **Сборка** | Vite 7 |
| **Управление состоянием** | Pinia |
| **Маршрутизация** | Vue Router 4 |
| **HTTP-клиент** | Axios |
| **Тестирование** | Jest + Vue Test Utils + Testing Library |
| **Линтинг** | ESLint (конфиг `plugin:vue/vue3-essential`) |
| **Стилизация** | Sass (SCSS) |
| **Транспиляция** | Babel |
| **Дополнительно** | TypeScript  |

## 🚀 Запуск проекта

### Требования

- Node.js 18+ (проверить: `node -v`)
- npm 9+ (или yarn / pnpm)

### Установка и запуск

# 1. Клонируйте репозиторий
git clone https://github.com/lyubashaten/vk-marusya.git
cd vk-marusya

# 2. Установите зависимости
npm install

# 3. Запустите в режиме разработки
npm run dev    |


✅ Реализованный функционал (задачи)

1. Главная страница (дашборд)
Отображение случайного фильма с обложкой и краткой информацией.
Кнопка для получения нового случайного фильма (без перезагрузки страницы).
Блок «Топ-10 фильмов» по IMDb-рейтингу (данные с API).

2. Страница со списком жанров
GET-запрос к API для получения списка жанров.
Каждый жанр оформлен в виде карточки с названием и изображением (кастомные иконки).
Клик по карточке → переход на страницу фильмов выбранного жанра.

3. Страница конкретного жанра
Отображение карточек фильмов, отфильтрованных по выбранному жанру.
Бесконечная подгрузка (infinite scroll): сначала 10 фильмов, при скролле – следующие 10, пока не закончатся.
Сортировка фильмов по рейтингу (по умолчанию).

4. Страница фильма (детальная)
Полная информация: обложка, название, рейтинг, описание, год выпуска, жанры, длительность и т.д. (согласно API).
Кнопка «Смотреть трейлер» – открывает модальное окно со встроенным видео (YouTube-iframe).
Кнопка «В избранное» / «Удалить из избранного» – состояние кнопки меняется в зависимости от наличия фильма в избранном.

5. Авторизация и регистрация
Модальное окно с двумя формами: вход и регистрация (переключение по кнопке).
Валидация полей: все поля обязательны, пустые поля подсвечиваются красной рамкой.
POST-запросы к /api/login и /api/register.
Сессионная авторизация с использованием cookie (withCredentials: true в Axios).
После успешного входа в шапке сайта вместо кнопки «Войти» отображается фамилия пользователя.
При обновлении страницы авторизация не сбрасывается (сессия восстанавливается).
Кнопка «Выйти» на странице аккаунта → GET-запрос на логаут, удаление сессии и cookie, редирект на главную.

6. Избранное (доступно только авторизованным)
Добавление/удаление фильмов в избранное через отдельные API-запросы.
Если неавторизованный пользователь пытается добавить фильм в избранное – автоматически открывается модальное окно авторизации.
Список избранных фильмов отображается на странице аккаунта.
При обновлении страницы избранное сохраняется (синхронизация с API).

7. Поиск фильмов
Поле поиска в шапке сайта → при клике открывается модальное окно.
Ввод названия фильма → GET-запрос к API, отображение результатов (карточки фильмов).
Возможность перейти на страницу любого найденного фильма.

8. Страница аккаунта пользователя
Отображение личной информации: фамилия, имя, email (данные из профиля).
Список фильмов, добавленных в избранное (с возможностью перехода на страницу фильма).
Кнопка «Выйти» для завершения сессии.

9. Общие технические требования
Проект написан на Vue 3 + TypeScript (с Composition API).
Управление состоянием: Pinia (хранилища пользователя, фильмов, жанров, избранного).
Маршрутизация: Vue Router 4 (защита маршрутов для авторизованных пользователей).
HTTP-запросы: Axios с настройкой withCredentials: true для работы с сессионными cookie.
Разделение на логические директории: pages/, components/, api/, store/, router/, assets/.
Соблюдение стайлгайда и принципов чистого кода:
длина методов ≤ 20 строк,
вложенность ≤ 3 уровней,
максимум 2-3 параметра у методов,
осмысленные имена переменных,
методы-глаголы, компоненты-существительные.
Обработка ошибок API: вывод понятных уведомлений (например, «Неверный логин или пароль»).
Прелоадеры (спиннеры) при асинхронных операциях (загрузка фильмов, авторизация, поиск).
Адаптивная вёрстка по макету Figma (Pixel Perfect).
Покрытие тестами (Jest + Vue Test Utils) наиболее сложных частей (опционально).
